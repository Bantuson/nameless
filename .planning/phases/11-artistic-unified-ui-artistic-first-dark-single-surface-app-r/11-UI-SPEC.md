# Phase 11 — UI Design Contract

> Visual and interaction contract for the Artistic Unified UI. Generated by gsd-ui-researcher, verified by gsd-ui-checker. Every decision in 11-CONTEXT.md is LOCKED and pre-populated below; discretion areas are decided here and marked `[discretion]`.

**Design read:** a single-surface creative instrument for a solo music producer, used alone, at night, in a dim room, mid-idea. Dark is not an aesthetic choice here; it is the physical scene. The interface must feel the way Sonder / Brent Faiyaz sounds: moody, spacious, intimate, analog-warm. Landing is brand register (design IS the product); the home surface is product register (design SERVES the work). Dials: `DESIGN_VARIANCE 8 / MOTION_INTENSITY 6 / VISUAL_DENSITY 3`.

Exceptions:
- **44px** minimum hit area for every icon-only control (WCAG target size).

## Typography

Exactly 4 sizes, 2 weights. One family carries hierarchy through weight + scale contrast (ratio ≥ 1.25 between steps); the mono is reserved for musical metadata so numbers read as instrument readouts, never body copy.

| Role | Size | Weight | Line Height | Family / Notes |
|------|------|--------|-------------|----------------|
| Body | 15px | 400 | 1.55 | Hanken Grotesk. Max line length 65ch. |
| Label / Meta | 13px | 400 | 1.4 | Hanken Grotesk for labels; **JetBrains Mono for key/tempo/BPM/LUFS/time-ranges/IDs only.** Uppercase permitted only on ≤4-word labels, tracking ≤ 0.08em, max one per panel. |
| Heading | 20px | 600 | 1.25 | Hanken Grotesk. Panel titles, section heads. Letter-spacing -0.01em. |
| Display | clamp(40px, 6vw, 76px) | 600 | 1.05 | Hanken Grotesk. Landing wordmark/tagline only. Letter-spacing -0.02em (never tighter than -0.04em). `text-wrap: balance`. |

Rules:
- No third weight, no italic-as-decoration, no serif anywhere.
- `font-display: swap` with fallback `system-ui, -apple-system, "Segoe UI", Roboto, sans-serif` (and `ui-monospace, "Cascadia Code", Menlo, Consolas, monospace`).
- No ALL-CAPS body copy. No gradient text ever.

## Color

Palette evolves the existing `styles.css` DNA (dusty rose + muted amber over violet-warm ink) rather than replacing it; identity preservation wins. All values verified ≥ 4.5:1 for text roles on their backgrounds. No pure `#000` or `#fff` anywhere.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0B0A10` (deep ink, violet-warm cast) | App background, scene backdrop, veil scrim base |
| Secondary (30%) | `#14121A` panels / `#1B1824` raised cards / `#2A2533` hairline borders | Overlays, drawers, dock, inputs (`#0F0E15`) |
| Accent (10%) | `#D8A79F` (dusty rose) | RESERVED list below only |
| Destructive / Hard-block | `#E08D8D` (muted coral) | Error text + the 422 incomplete-attribution block state, failed jobs. There are NO destructive actions in this phase (API has no deletes); coral is error semantics only. |

**Accent reserved for (exhaustive):**
1. The landing CTA ("Enter the studio")
2. The primary home action (Capture) in the dock
3. The record button
4. The selected/active constellation node halo
5. Active job progress indicator (the "listening" pulse)

Nothing else. Links, secondary buttons, hover states use text-tone shifts and border lightening, not accent.

**Supporting semantic tokens (quiet, sub-10% each):**

| Token | Value | Meaning |
|-------|-------|---------|
| `--text` | `#ECE9F5` | Primary text (≈16:1 on ink) |
| `--text-muted` | `#A9A4BD` | Secondary text (≈8:1) |
| `--text-dim` | `#8B86A0` | Tertiary/meta (≈5.5:1, never below 13px) |
| `--amber` | `#C9B277` | Sampling/attribution provenance ONLY: `sampled` nodes, rights badges, credits rows |
| `--ok` | `#86D4AD` | `analyzed`/complete state ticks |
| `--focus` | `#9FB6FF` | Focus rings exclusively (2px ring + 2px offset), distinct from accent so accent stays reserved |

**Provenance → color mapping (constellation + badges):**

| Provenance | Node color |
|------------|-----------|
| `human_recorded` | `--text` warm-white core |
| `sampled` | `--amber` core |
| `derived` | `--text-dim` core |
| `ai_generated` (future M1) | reserved: cool `#9FB6FF` family; do not use elsewhere |

Node state ring: `captured/analyzing` = dim + breathing pulse; `analyzed` = full luminance + key/tempo label; `placed` = connected with lit lineage edge.

Banned: neon-on-black, outer glows on DOM elements (soft additive glow lives inside the WebGL scene only), gradient text, AI-purple, side-stripe borders, pure black.

## Surface Architecture (the single-surface contract)

**Hard rule:** exactly two app states, no routed pages. `landing` and `home`. Everything else is a layer over `home`. If a feature needs a URL, it is out of scope for this phase.

**Landing ↔ home relationship `[discretion]`:** one continuous world via a persistent WebGL canvas, implemented as a two-state shell. The Three.js canvas mounts once and never unmounts:

- **Landing:** scene at full intensity. Composition: wordmark "Nameless" (Display), one tagline line, one CTA. Max 3 text elements. Left-aligned composition over the scene (anti-center bias), scene weighted right.
- **Enter transition:** 900ms camera pull-back + scene dims to ~20% intensity; landing copy crossfades out (300ms), home surface fades in (500ms, 200ms delay). Under reduced motion: instant cut with 150ms crossfade.
- **Home:** the dimmed scene remains as the living background. The fragment constellation is the visual and functional center. One dock at the bottom. Nothing else is visible at rest.

**Progressive disclosure staging `[discretion]`:**

| Layer | z-token | Contents | Reveals on |
|-------|---------|----------|-----------|
| 0 scene | `--z-scene: 0` | WebGL nebula (aria-hidden) | always |
| 1 surface | `--z-surface: 10` | Constellation (or first-run state), project name top-left, quiet "List" view toggle | entering home |
| 2 dock | `--z-dock: 20` | 4 actions: **Capture** (accent, primary), References, Stems, Credits + project switcher | always in home |
| 3 overlay | `--z-overlay: 30` | ONE overlay at a time: Capture sheet, Fragment veil, Reference drawer, Stem drawer, Credits veil | user intent (dock click / node click) |
| 4 toast | `--z-toast: 40` | Status whispers (job progress, success, errors) | system events |

Z-index uses these five tokens only. No arbitrary values.

Overlay grammar (pick per component, consistent forever):
- **Drawer** (References, Stems): slides from the right, 420px wide (100% under 768px), scrim at `rgba(11,10,16,0.6)`.
- **Sheet** (Capture): rises from the dock, max-width 520px, centered on the dock's Capture button.
- **Veil** (Fragment detail, Credits): centered, max-width 640px, scene visible behind scrim.

All overlays: `Escape` closes, scrim click closes, focus is trapped and returned to the trigger. Only one open at once; opening another replaces with a 200ms crossfade. Depth-within-overlay (e.g. attribution form inside Stem drawer) advances in-place with a slide, with a visible back affordance.

**First-time user path (no tutorial, one obvious next action at every step):**
1. Landing → "Enter the studio".
2. No projects exist → the surface shows the Start-a-project moment (single input + CTA, copy below). No dock until a project exists.
3. Empty constellation → the empty-state whisper points at Capture; Capture is the only accent element on screen.
4. Capture → node appears pulsing ("listening") → resolves to analyzed with key/tempo. The loop is: capture → see it understood → sample → credits, each step's completion visually pointing at the next.

**AI-native rules (locked):** intent note is a first-class field on capture, never buried. Summaries over raw data everywhere: vibe prose leads the reference card, measurables follow; the contract's types cannot carry arrays and the UI must not fake them. The system narrates what it is doing in human language (status whispers below), never job IDs first.

## Three.js Scene Contract `[discretion]`

**Concept: "Night Bloom."** A slow-breathing particle nebula: 6,000–10,000 GPU-buffered points drifting in a loose disc under curl-noise-like flow, colored along an ink → dusty-rose → muted-amber ramp (scene may use additive blending + soft sprite glow; this is the ONLY place glow exists). A gentle luminance pulse travels the field at 72 BPM (1.2 Hz) so the scene has tempo without sound. Depth fog fades edges into `#0B0A10` so canvas and page are one surface.

| Contract item | Requirement |
|---------------|-------------|
| Tech | `three` + `@react-three/fiber`; scene component lazy-loaded (`React.lazy`), landing shows the CSS fallback until ready |
| Fallback (mandatory, designed) | No WebGL / load failure / test env: static radial-gradient backdrop (`#181826` bloom at 70% -10% over ink, matching current styles.css) + fine noise grain on a `pointer-events:none` fixed pseudo-element. Never a blank or broken hero. |
| Interaction | Pointer parallax: camera target eases toward pointer with critically-damped smoothing (motion-value pattern, never React state per frame). No scroll listeners of any kind. |
| Audio-reactive port | `SceneDriver` interface `{ amplitude(): number; pulse(): void }` injected via props/context with a default idle driver. M2 supplies a real audio driver; this phase ships the port + idle impl + a test fake (DI law). |
| State coupling | Scene intensity: 1.0 on landing, 0.2 in home. Capture success calls `pulse()` once (a visible bloom ripple acknowledges the capture). |
| Performance | DPR clamped to `min(devicePixelRatio, 1.75)`; RAF paused on `document.hidden`; geometry/material disposed on unmount; particle count halves below 4-core/`deviceMemory < 4` heuristics. Target 60fps desktop. |
| Reduced motion | `prefers-reduced-motion: reduce` → render a single static frame (no RAF loop), parallax off, pulse becomes a 150ms opacity blink. |
| Accessibility | `<canvas aria-hidden="true">`; the scene carries zero information that is not also in DOM. |

## Constellation Contract (the heart of home)

The fragment graph is a living constellation, not a data table, and not inside WebGL: it renders as an **SVG layer** over the dimmed scene so it stays accessible, unit-testable in jsdom, and crisp.

| Contract item | Requirement |
|---------------|-------------|
| Data | `getProjectGraph(projectId)` nodes + edges. Poll every 5s while any node is `captured/analyzing` (until real push exists). |
| Layout | Deterministic pure function `layoutConstellation(nodes, edges, seed=projectId) → positions` (radial/organic placement seeded by fragment id; same input, same sky). Pure = unit-testable. |
| Node anatomy | 12px core disc + 28px soft halo. Core color by provenance (table above), ring by state. Analyzed nodes show `note` (truncated 24ch, Body 15px) and `key · tempo` beneath in JetBrains Mono 13px (e.g. `A:min  92 BPM`). |
| Edges | Lineage (`parent_fragment_id`) as 1px curved hairlines at `--border-strong`, brightening when either endpoint is hovered/selected. |
| Interaction | Every node is a real `<button>` (tabbable in creation order). Click/Enter opens the Fragment veil. Hover/focus lifts halo luminance 20%. Selected node gets the accent halo. |
| Analyzing motion | 2.4s opacity breath (0.5 ↔ 0.9). Reduced motion: static 0.7 + textual "listening" badge. |
| List alternative | A quiet "List" toggle (top-right of surface) swaps the constellation for an accessible list of the same nodes (id, note, kind, state, key/tempo). Same data source, same click target. This is also the a11y/testing fallback. |
| Empty state | The dimmed scene IS the empty sky; centered whisper copy (see Copywriting) with the arrow of intent pointing at the dock's Capture. |

Every component must ship all four states: **loading** (skeleton matching final shape, no spinners), **empty** (composed, with next step), **error** (problem + path forward), **success/idle**. Buttons get `:active` tactile press (`scale(0.98)`), one primary per overlay.

| Component | API methods covered | Notes |
|-----------|--------------------|-------|

| Project switcher (top-left) | `listProjects`, `createProject` | Quiet text button → small popover: project list + "New project" input. |
| Dock | — | 4 labeled icon actions + switcher. Single row, ≤72px tall. |
| Capture sheet | `capture` | Record (MediaRecorder) OR drop/browse file; intent note (required, textarea); kind select (9 kinds, plain labels); submit. Recording state shows elapsed time in mono. |
| Constellation / List | `getProjectGraph`, `listFragments` | See Constellation Contract. |
| Fragment veil | `getFragment` (+ `getSample` when `provenance=sampled`) | Note as the headline; kind/provenance/state badges; `key · tempo · duration` in mono; lineage line ("derived from …") when parented; sampled fragments show full attribution block + rights badge in amber. |
| Reference drawer | `listReferences`, `uploadReference`, `getReferenceSummary`, `attachReference` | List → detail-in-drawer. **The 422 `incomplete_attribution` state (designed, mandatory):** on `IncompleteAttributionError`, the Attribution form stays open; an inline coral banner appears above the submit button; each missing field's label turns coral with a per-field "required for the credit" line; focus moves to the first missing field. The banner explicitly states nothing was created. Field-name humanization: `source_track`→"source track", `stem`→"stem", `source_title`→"title", `source_artist`→"artist", `stem_type`→"stem type", `time_range`→"time range", `rights`→"rights status".

## Motion Contract

Ease: `cubic-bezier(0.16, 1, 0.3, 1)` (ease-out-quint family) for everything. No bounce, no elastic, no infinite DOM loops except the analyzing breath. Animate `transform`/`opacity` only.

| Motion | Duration | Notes |
|--------|----------|-------|
| Hover/press micro | 150–200ms | `:active scale(0.98)` |
| Overlay in / out | 420ms / 260ms | Drawer slide, sheet rise, veil fade+scale(0.98→1) |
| Landing → home | 900ms scene + 500ms content crossfade (200ms delay) | See Surface Architecture |
| Analyzing node breath | 2.4s loop | Opacity 0.5 ↔ 0.9 |
| Whisper in/out | 260ms | Slide+fade from bottom-left |
| Capture bloom ripple | ≤1s, once | `SceneDriver.pulse()` |

`prefers-reduced-motion: reduce` (non-negotiable): all transitions collapse to ≤150ms crossfades or instant; breath becomes static + text badge; scene renders one static frame; parallax off. Banned: `window.addEventListener('scroll', …)`, RAF loops touching React state, layout-property animation, uniform entrance animation applied to every element.

## Accessibility Contract

- WCAG AA minimum everywhere (values above pre-verified); focus ring `2px solid var(--focus)` + 2px offset on every interactive element, never removed.
- Skip link retained. Landmarks: `header` (project switcher), `main` (surface), `nav` (dock), `dialog` semantics (`role="dialog"`, `aria-modal`, labelled by panel title) for every overlay.
- Overlays: focus trap, `Escape` closes, focus returns to trigger.
- Canvas `aria-hidden`; constellation nodes are real buttons; the List toggle is the equivalent-experience guarantee.
- Forms: label above input, error text below input and referenced via `aria-describedby`; 422 moves focus to first missing field.
- Live regions: whispers in an `aria-live="polite"` region; error banners `role="alert"`.
- All functionality keyboard-complete: the entire core loop (capture → understand → sample → credits) must be operable without a pointer.

## Anti-Slop Guardrails (binding for this phase)

Beyond the skills' global bans, these are the ones this build is most at risk of:
1. No card grids of identical tiles; group with hairlines, spacing, and the constellation itself.
2. No uppercase tracked eyebrows above sections; panel titles stand alone.
3. No em dashes rendered anywhere.
4. No spinners; skeletons match final shape.
5. No fake data in empty states; empty states are composed copy, not grayed-out fake rows.
6. No glow/glassmorphism on DOM; atmosphere lives in the scene layer.
7. Corner radius lock: cards/panels 12px, inputs 8px, pills full. Nothing above 16px.
8. Shadows, when used, tinted to ink (`rgba(5,4,10,…)`), blur ≤ 24px on overlays only, never paired with a 1px border as decoration.
9. If it would look templated in a screenshot, it fails the phase's own success criterion (CONTEXT: "templated/bootstrap-looking output is a failure condition").

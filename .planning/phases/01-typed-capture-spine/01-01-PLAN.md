<objective>
Deliver the **Walking Skeleton** of the Nameless control plane: a Cargo workspace whose `nameless` CLI can, under a `--local` profile with NO Postgres, capture an audio fragment with an intent note into a project, store the raw audio immutably by content hash, and list it back by ID — compact-by-default (never echoing audio bytes or feature arrays).

This is the thinnest end-to-end slice that proves the spine. It establishes the ports-and-adapters seam (`ObjectStore`, `FragmentRepo`) that makes every later phase RAM-safe-verifiable: the production heavy leaf (S3/R2, Postgres) and the local fake satisfy the same trait, so local verification exercises real control flow with only the heavy leaf swapped.

Purpose: prove capture→storage→list works end-to-end on the 4GB machine and lock the architectural decisions (workspace layout, content-hash storage, feature-gated heavy deps) the rest of the project builds on. See SKELETON.md.
Output: a compiling 3-crate workspace; `FilesystemObjectStore`, `InMemoryFragmentRepo`, `FileFragmentRepo`, symphonia probe; `nameless project create | capture | fragments list | fragments show` wired to `--local`.
</objective>

    `object_store_fs.rs` — `FilesystemObjectStore { root: PathBuf }` implementing `ObjectStore`:
    - `put(key, bytes)`: write bytes to `root/<key>` only if absent (immutable — never overwrite an existing key); create parent dirs. `key` is the SHA-256 hex of the bytes (content addressing), so traversal is impossible (key is hex, not user input).
    - `get`/`exists` read by key. Provide a free helper `content_hash(bytes: &[u8]) -> String` (lowercase hex SHA-256) used by the CLI to derive the key. This filesystem impl is the fake that the S3/R2 impl (Plan 04) mirrors — same trait, swapped leaf.

    `repo_mem.rs` — `InMemoryFragmentRepo` backed by `Mutex<HashMap>` (projects + fragments). Pure RAM, for unit/state-machine tests (no DB server).

    `repo_file.rs` — `FileFragmentRepo { path: PathBuf }`: persists projects+fragments as a single `serde_json` document on disk (load-modify-store under a file lock or read/rewrite). This is the `--local` persistence so `capture` then a SEPARATE `fragments list` process invocation sees the row (CONTEXT: "runs WITHOUT Postgres"). Chosen over SQLite to keep the default compile pure-Rust and minimal (Claude's discretion per CONTEXT); document this.

    `probe.rs` — `fn probe(bytes: &[u8]) -> ProbeResult` using symphonia: detect format, return `{ duration_ms: Option<u32>, sample_rate: Option<u32> }`. On unsupported/garbage input return `None`s (do NOT fail capture — store bytes regardless, per CONTEXT). Never decode the whole stream into memory beyond what symphonia needs for the header/metadata (RAM discipline).

    Add unit tests in each file (see acceptance criteria).

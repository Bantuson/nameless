<objective>
Add a no-API extraction path to the knowledge-pipeline: a `FileClaimExtractor` adapter that reads
pre-mined claims JSON per video from a directory, plus a `claims mine --mined-dir <dir>` CLI plane
that wires it to the REAL `FilesystemCorpusStore` + REAL `SqliteClaimStore` through the UNCHANGED
`MiningPipeline`.

Purpose: the user has no Anthropic API credits. Claude Code (in-session) mines claims file-to-file
and drops `{video_id}.json` files into a directory; the pipeline then ingests them so
`verify_citation`, dedup, and cross-reference judge them exactly like API output. The gate stays
identical — only the extraction leaf changes.

Output: new adapter module + eager export, one new CLI plane + `--mined-dir` flag, one new test file.
`MiningPipeline`, `verify_citation`, and the citation gate are NOT modified.
</objective>

<design_notes>
Load-bearing facts discovered during planning — do not re-derive, do not deviate:

1. **Error seam.** `MiningPipeline.mine` (mining_pipeline.py, target loop) already wraps
   `self._extractor.extract(...)` in `try/except Exception` and records
   `MineOutcome(detail=f"extract error: {exc}")`, then continues. So the adapter expresses
   "missing file — skip" and "malformed JSON — loud" by RAISING precise exceptions; the UNCHANGED
   pipeline converts them into per-video report lines and the run continues. This is why the
   constraint "do not modify MiningPipeline" is satisfiable for free.
2. **Payload shape.** The JSON file body is exactly the `emit_claims` tool input that
   `EXTRACTION_TOOL_SCHEMA` describes: a top-level object with a `claims` array of
   `{claim_text, technique, stage, genre?, stance?, timestamp_ms, quote, confidence}` entries.
   `parse_extractor_output(raw, transcript, genres=...)` (pure, already tested) validates each
   entry via `_RawClaimIn`, re-anchors the quote's timestamp to the real segment, and binds
   `source_video_id` + `caption_source` from the transcript — the file can never poison identity
   or citation fields. Reuse it verbatim; invent nothing.
3. **Return contract parity.** `AnthropicClaimExtractor.extract` returns
   `parse_extractor_output(raw, transcript, genres=list(genres))` and returns `[]` on
   `not transcript.segments`. Mirror both behaviors exactly.
4. **Path safety.** `video_id` becomes a filesystem path component (`{video_id}.json`) and can
   arrive from the `--video` CLI argument — the exact P3 CR-01 traversal shape. `corpus_fs.py`
   already has the guard: `_safe_video_id` (regex `^[A-Za-z0-9_-]{1,64}$`, raises `ValueError`).
   Import and reuse it from `.corpus_fs`; do NOT write a second guard that can drift.
5. **Eager import is safe.** The adapter needs only stdlib (`json`, `pathlib`) + the pure schema
   module — so it joins the eagerly-imported family in `adapters/__init__.py` (unlike the
   anthropic adapter, which stays lazy/unexported).
6. **This is one of the rare directly-runnable-here changes**: tests use fakes/tmpdir/sqlite only.
   Verify steps MUST actually run `uv run pytest` (from `knowledge-pipeline/`).
</design_notes>

| `--video` CLI arg → filesystem path | User-supplied string becomes a path component (`{video_id}.json`) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-q260706-01 | Tampering | FileClaimExtractor path build | mitigate | Reuse `_safe_video_id` from corpus_fs (P3 CR-01 guard) before any path composition; traversal test in Task 1 |
| T-q260706-02 | Spoofing | Claim identity/citation fields | mitigate | `parse_extractor_output` binds `source_video_id`/`caption_source` from the transcript and re-anchors timestamps — file content cannot set them; unchanged `verify_citation` still judges every quote against the real snapshot |
| T-q260706-03 | DoS/robustness | Malformed/hostile JSON input | mitigate | `json.JSONDecodeError` → loud `ValueError` naming the file; per-video pipeline catch bounds blast radius to one target; pydantic `_RawClaimIn` validation skips bad entries |
| T-q260706-04 | Elevation | New dependencies | accept | None added — adapter is stdlib + existing pure code; no package installs in this plan |
</threat_model>

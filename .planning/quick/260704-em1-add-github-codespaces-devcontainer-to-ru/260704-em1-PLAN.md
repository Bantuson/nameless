  <name>Task 2: Create .devcontainer/post-create.sh with marker-guarded migrations</name>
  <files>.devcontainer/post-create.sh</files>
  <action>
Create `.devcontainer/post-create.sh` — bash, `set -euo pipefail`, idempotent end to end (post-create can re-run on container rebuilds against the persisted db volume). Sections in order:

1. **System packages:** `sudo apt-get update &amp;&amp; sudo apt-get install -y --no-install-recommends postgresql-client ffmpeg` (apt is idempotent by nature).

2. **uv:** install via the official Astral installer `curl -LsSf https://astral.sh/uv/install.sh | sh`, guarded by `if ! command -v uv ...` (also check `$HOME/.local/bin/uv` since the installer targets `~/.local/bin`, which the devcontainer base puts on PATH).

3. **Wait for Postgres:** loop up to 30 attempts, 2s apart: `pg_isready -h db -p 5432 -U nameless` (host `db`, matching the compose service). Fail loudly with a clear message if it never comes up.

4. **Migrations — marker-table strategy (the honest one, per verified_facts: 0001/0003/0004/0005 are NOT re-runnable):**
   - `psql "$DATABASE_URL" -v ON_ERROR_STOP=1 -c "create table if not exists _devcontainer_migrations (filename text primary key, applied_at timestamptz not null default now())"`
   - For each `f` in `migrations/*.sql` (shell glob = filename order, same as ci.yml):
     - Query the marker table for `basename "$f"`; if present, echo "skip (already applied)" and continue.
     - Else `psql "$DATABASE_URL" -v ON_ERROR_STOP=1 -f "$f"` then insert the basename into `_devcontainer_migrations`.
   - Rationale to state in a header comment: a fresh-vs-absent check on one table (e.g. fragments) would break the moment a NEW migration file lands — the per-file marker applies only what is missing. This intentionally diverges from ci.yml's blind loop because CI gets a fresh database every run and this volume persists.

5. **Web deps:** `npm ci --prefix web` (or `cd web &amp;&amp; npm ci`) — `npm ci` is idempotent (clean-installs from web/package-lock.json, same as ci.yml's web job).

6. Print a short "ready" summary naming the two run commands (server + web) so they're visible in the creation log.

Make the file executable-style (shebang `#!/usr/bin/env bash`); invoked via `bash .devcontainer/post-create.sh` so the executable bit is not load-bearing on Windows checkouts.
